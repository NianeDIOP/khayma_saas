Vérification de l'environnement XAMPP

PHP 8.3 · Composer · Node.js · npm · Extensions PostgreSQL

Validé
XAMPP doit être installé. Ouvre un terminal PowerShell dans le dossier de XAMPP pour exécuter les commandes ci-dessous.
  • Vérifier PHP : php -v → doit afficher PHP 8.3+ cfg
  • Vérifier Composer : composer -V → doit afficher Composer 2.x cfg
  • Vérifier Node.js : node -v → v20+ et npm -v → 10+ fe
  • Dans C:\xampp\php\php.ini : décommenter extension=pdo_pgsql et extension=pgsql cfg
  • Vérifier aussi que extension=zip et extension=fileinfo sont activées dans php.ini cfg
  • Redémarrer Apache XAMPP puis confirmer : php -m | findstr pgsql → doit retourner pdo_pgsql et pgsql test

composer -V
node -v
npm -v
php -m | findstr pgsql

Configuration PostgreSQL local

Créer la base khayma_dev et l'utilisateur dédié

Validé
PostgreSQL doit être démarré. Vérifie dans les Services Windows que postgresql-x64-16 est en cours d'exécution.
  • Ouvrir psql : psql -U postgres db
  • Créer l'utilisateur : CREATE USER khayma_user WITH PASSWORD 'khayma_secret_2024'; db
  • Créer la base : CREATE DATABASE khayma_dev OWNER khayma_user ENCODING 'UTF8' TEMPLATE template0; db
  • Accorder les privilèges : GRANT ALL PRIVILEGES ON DATABASE khayma_dev TO khayma_user; db
  • Tester la connexion : psql -U khayma_user -d khayma_dev -c "SELECT version();" test 
-- Dans psql (connecté en tant que postgres) :
CREATE USER khayma_user WITH PASSWORD 'khayma_secret_2024';
CREATE DATABASE khayma_dev
  OWNER khayma_user
  ENCODING 'UTF8'
  TEMPLATE template0;
GRANT ALL PRIVILEGES ON DATABASE khayma_dev TO khayma_user;
\q

-- Test connexion :
psql -U khayma_user -d khayma_dev -c "SELECT version();"

Création du projet Laravel 11

Installation, configuration .env PostgreSQL, migration initiale

Validé
Le projet sera créé dans C:\xampp\htdocs\khayma\. Le dossier Saas\ (où tu es maintenant) est uniquement pour ce workflow.
  • Créer le projet Laravel : cd C:\xampp\htdocs puis composer create-project laravel/laravel khayma be
  • Entrer dans le dossier et copier l'env : cd khayma puis copy .env.example .env cfg
  • Configurer le fichier .env : DB_CONNECTION=pgsql, DB_PORT=5432, DB_DATABASE=khayma_dev, DB_USERNAME=khayma_user, DB_PASSWORD=khayma_secret_2024 cfg
  • Générer la clé d'application : php artisan key:generate be
  • Lancer les migrations par défaut : php artisan migrate → doit créer les tables users, cache, jobs test
  • Démarrer le serveur et vérifier : php artisan serve → http://localhost:8000 doit afficher la page d'accueil Laravel test 
cd C:\xampp\htdocs
composer create-project laravel/laravel khayma
cd khayma
copy .env.example .env

# Dans .env, modifier :
DB_CONNECTION=pgsql
DB_HOST=127.0.0.1
DB_PORT=5432
DB_DATABASE=khayma_dev
DB_USERNAME=khayma_user
DB_PASSWORD=khayma_secret_2024

php artisan key:generate
php artisan migrate
php artisan serve

Installation Frontend — Vue 3 · Inertia.js · Tailwind CSS

Stack frontend complet avec Vite comme bundler

Validé
  • Installer Inertia côté serveur : composer require inertiajs/inertia-laravel be
  • Publier le middleware HandleInertiaRequests : php artisan inertia:middleware et l'enregistrer dans bootstrap/app.php be
  • Installer les dépendances npm frontend : npm install @inertiajs/vue3 vue @vitejs/plugin-vue fe
  • Installer Tailwind CSS v4 : npm install -D tailwindcss @tailwindcss/vite fe
  • Configurer vite.config.js avec les plugins Vue et Tailwind (voir code ci-dessous) fe
  • Créer resources/js/app.js avec l'initialisation Inertia + Vue 3 (voir code ci-dessous) fe
  • Tester la compilation sans erreur : npm run dev test 
composer require inertiajs/inertia-laravel
php artisan inertia:middleware
npm install @inertiajs/vue3 vue @vitejs/plugin-vue
npm install -D tailwindcss @tailwindcss/vite
npm run dev
// vite.config.js
import { defineConfig } from 'vite';
import laravel from 'laravel-vite-plugin';
import vue from '@vitejs/plugin-vue';
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
  plugins: [
    laravel({ input: ['resources/js/app.js'], refresh: true }),
    vue(),
    tailwindcss(),
  ],
});
// resources/js/app.js
import { createApp, h } from 'vue';
import { createInertiaApp } from '@inertiajs/vue3';
import { resolvePageComponent } from 'laravel-vite-plugin/inertia-helpers';
import '../css/app.css';

createInertiaApp({
  resolve: name => resolvePageComponent(
    `./Pages/${name}.vue`,
    import.meta.glob('./Pages/**/*.vue')
  ),
  setup({ el, App, props, plugin }) {
    createApp({ render: () => h(App, props) })
      .use(plugin)
      .mount(el);
  },
});

Initialisation Git & Premier commit

Versionner le projet et pousser sur GitHub

Validé
Ne jamais committer le fichier .env — vérifier qu'il est bien listé dans .gitignore avant le premier commit.
  • Initialiser Git : git init cfg
  • Vérifier que .env, node_modules/ et vendor/ sont bien dans .gitignore cfg
  • Créer un dépôt privé khayma sur GitHub et copier l'URL SSH cfg
  • Ajouter tous les fichiers et faire le premier commit : git add . puis git commit -m "init: Laravel 11 + Vue 3 + Inertia + Tailwind + PostgreSQL" cfg
  • Lier le remote et pousser : git remote add origin git@github.com:TON_USER/khayma.git puis git push -u origin main cfg 
git init
git add .
git commit -m "init: Laravel 11 + Vue 3 + Inertia + Tailwind + PostgreSQL"
git remote add origin git@github.com:TON_USER/khayma.git
git branch -M main
git push -u origin main
