Structure des dossiers Laravel

Créer l'architecture en couches : Services, Repositories, Enums, Traits

En cours
L'architecture en couches sépare les responsabilités : Controllers (HTTP) → Services (logique métier) → Repositories (accès données). Cela facilite les tests et la maintenabilité.
  • Créer app/Services/ — couche logique métier dev
  • Créer app/Repositories/ — couche accès données dev
  • Créer app/Enums/ — constantes typées (rôles, statuts, modules) dev
  • Créer app/Traits/ — comportements réutilisables (HasTenant, HasAudit) dev
  • Créer app/Http/Requests/ — Form Requests centralisés dev
  • Créer app/Http/Resources/ — API Resources (transformateurs JSON) dev 
mkdir app/Services app/Repositories app/Enums app/Traits
mkdir app/Http/Requests app/Http/Resources

1.3

Configuration Laravel Sanctum

Auth SPA + tokens API pour l'espace entreprise et le backoffice

À faire
  • Publier la config Sanctum : php artisan vendor:publish --provider="Laravel\Sanctum\SanctumServiceProvider" cfg
  • Ajouter le trait HasApiTokens au modèle User dev
  • Configurer sanctum.stateful dans config/sanctum.php pour localhost cfg
  • Exécuter php artisan migrate — migrer la table personal_access_tokens cfg
  • Tester : créer un token de test via Tinker et vérifier qu'il s'enregistre en DB test 
php artisan vendor:publish --provider="Laravel\Sanctum\SanctumServiceProvider"
php artisan migrate
1.4

Configuration Spatie Permission (RBAC)

Rôles : super_admin, owner, manager, caissier, magasinier

À faire
  • Publier la config : php artisan vendor:publish --provider="Spatie\Permission\PermissionServiceProvider" cfg
  • Ajouter les traits HasRoles au modèle User dev
  • Exécuter php artisan migrate — tables roles, permissions, model_has_roles... cfg
  • Créer le seeder RolesAndPermissionsSeeder — rôles : super_admin, owner, manager, caissier, magasinier dev
  • Exécuter php artisan db:seed --class=RolesAndPermissionsSeeder et vérifier en DB test 
php artisan vendor:publish --provider="Spatie\Permission\PermissionServiceProvider"
php artisan migrate
php artisan db:seed --class=RolesAndPermissionsSeeder
